❗️ За один пост нормально покрыть тему тестирвоания блоков не получится. Больше информации по возможностям
[bloc_test](https://pub.dev/packages/bloc_test#unit-test-with-bloctest) можно найти в
[документации](https://pub.dev/packages/bloc_test), а больше примеров конкретно по
[bloc_example](https://github.com/fluttermiddlepodcast/bloc_example) в
[репозитории](https://github.com/fluttermiddlepodcast/bloc_example) проекта.

Последний официальный плагин для [bloc](https://pub.dev/packages/bloc) и
[flutter_bloc](https://pub.dev/packages/flutter_bloc), который мы не разобрали.

[bloc_test](https://pub.dev/packages/bloc_test) - это пакет для тестирования блоков и кубитов. Он содержит готовые
методы
для написания тестов для бизнес-компонентов.

В нашем проекте для тестирования [bloc_example](https://github.com/fluttermiddlepodcast/bloc_example) теперь есть тесты,
они уже находятся в [мастере](https://github.com/fluttermiddlepodcast/bloc_example/blob/master/test).

<u>Написал тесты для следующих кейсов:</u>

— [Инициализация, постраничная загрузка и загрузка последней страницы с данными для экрана с
пользователями](https://github.com/fluttermiddlepodcast/bloc_example/blob/master/test/features/users/users_bloc_test.dart).

— [Инициализация, загрузки, logIn и logOut
профиля](https://github.com/fluttermiddlepodcast/bloc_example/blob/master/test/features/profile/profile_bloc_test.dart).

<u>Помимо создания тестов для блоков, было сделано:</u>

— Конфиг для тестов в [GitHub Actions](https://github.com/features/actions). Сам файл
[тут](https://github.com/fluttermiddlepodcast/bloc_example/blob/master/.github/workflows/flutter.yaml), внизу будет
секция `tests`.

— Моки для [hydrated_bloc](https://t.me/fluttermiddlepodcast/426) и репозиториев из проекта. Реализованы через
[mockito](https://pub.dev/packages/mockito).

— Обновление Флаттера и рефакторинг под новые синтаксические фичи Дарта.

**<u>Ознакомиться со всеми изменениями можно по
[ссылке](https://github.com/fluttermiddlepodcast/bloc_example/tree/master/test).</u>**
